Connect Your GitHub Repo

One-time setup · 3 minutes · everything lives in git

Your trades are stored in data/trades.json inside your GitHub repo. The journal reads and writes it directly via the GitHub API — no database needed.
1
Create a private GitHub repo (e.g. tradelog). Push the files from the ZIP — or just create the repo and let the bridge script do the first commit.
2
Generate a Personal Access Token: GitHub → Settings → Developer Settings → Personal Access Tokens → Fine-grained → New Token. Give it Read and Write access to Contents on your repo. Copy the token.
3
Enable GitHub Pages on your repo: Settings → Pages → Source = main branch → / (root). Your journal URL will be https://yourusername.github.io/tradelog/.
4
Fill in the fields below, then click Save & Connect.
For private GitHub repos, GitHub Pages is only available on paid plans. If you want free hosting, use Cloudflare Pages instead — it's free and connects to private repos. See the Git tab in the journal for instructions.

Option A — GitHub Pages (simplest)
1
Push the repo. Go to Settings → Pages → Source: main branch, / root.
2
Your journal lives at https://<you>.github.io/tradelog/
⚠
GitHub Pages is public for free accounts. Secure it with the journal PIN. For private Pages, you need GitHub Pro ($4/mo) — or use Option B.
Option B — Cloudflare Pages (free + private)
1
Go to Cloudflare Dashboard → Pages → Connect to Git. Pick your private repo.
2
Build settings: Framework = None, Build output = /. Deploy.
3
Enable Cloudflare Access (free): Zero Trust → Access → Applications → add your Pages URL. Restricts access to email you allow. Auto-deploys on every git push — no manual step.
